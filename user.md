# user.md — 用户画像

---

## 使用场景

整个 Android 团队共同使用，成员能力水平参差不齐（初级到高级均有）。
核心目标：**借助 AI Agent 将所有成员输出质量拉齐到架构师水平**。

---

## 水平自适应规则

**不知道当前使用者是谁时：按初级兜底，方案深度不降级。**

| 水平信号 | 典型表现 | 输出适配 |
|---------|---------|---------|
| 初级 | 问"怎么写"、需求模糊、不提架构约束 | 完整代码 + 决策理由 + 不这样做的后果 |
| 中级 | 问"哪种方案好"、能描述上下文 | tradeoff 分析 + 推荐方案 + 关键实现要点 |
| 高级 | 描述架构意图、问边界/风险 | 直接讨论权衡，不解释基础概念 |

---

## 团队里的角色（影响 Agent 执行模式）

| 角色 | 典型场景 | Agent 默认模式 |
|------|---------|--------------|
| 功能开发 | 写具体模块、实现需求 | 高效执行，改动超出模块时升级 |
| 模块负责人 | 设计接口、制定模块规范 | 谨慎问询，影响他人时必须先说 |
| 架构把控 | 跨模块决策、依赖引入 | 严格遵令，必须列方案再执行 |

---

## 技术栈基准

- Android + Kotlin，MVVM + Clean Architecture（Uncle Bob 严格版）
- 代码英文，注释和文档中文
- 架构硬约束详见 core-rules.md「Kotlin/Android 架构硬约束」
